## moritakuが実装したロジックの全体像

共通アーキテクチャ: 「GitHub APIで情報を取得 → Gemini APIでJSON構造化 → 初心者向けUIに変換」

### 4つのAIパイプライン

| # | パイプライン | 入力 | AI処理 | 出力 |
|---|---|---|---|---|
| ① | リポジトリ分析 | repoUrl | Octokitでファイル構造・README・全Issue取得 → Geminiで分析 | 技術スタック、学習ロードマップ、全Issue分解 |
| ② | タスク分解 | Issue URL | GitHub APIでIssue取得 → Geminiで分解 | description、前提知識リスト、極小ステップ |
| ③ | オンボーディング生成 | projectName, repoUrl | Geminiでガイド生成 | 環境構築Markdownガイド |
| ④ | AIコードレビュー | diff or PR URL | GitHub APIでdiff取得 → Geminiでレビュー | critical/suggestions/good のJSON |

## 原稿（30秒）

> StepByの裏側のロジックについて説明します。
> 共通のアーキテクチャとして、GitHub APIやOctokitでリポジトリの情報を取得し、
> Gemini APIに渡してJSON形式で構造化データを出力させ、
> それを初心者向けのUIに変換する、という流れで統一しています。
> この仕組みで4つの機能を実現していて、
> リポジトリ全体の分析とIssue一括分解、個別Issueの極小タスク分解、
> 環境構築ガイドの自動生成、そしてAIコードレビューです。
> すべてGemini 2.5 Flashで動いています。
